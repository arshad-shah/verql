Query Editor Design

The SQL editing experience — how writing and running queries should feel

A

Monaco-Powered Editor

Use VS Code's Monaco editor as the SQL editing surface. Get syntax highlighting, autocomplete, multi-cursor, minimap, and bracket matching for free. Add custom SQL language services on top for table/column autocomplete.

  • Full VS Code editing experience — familiar to every developer
  • Schema-aware autocomplete (tables, columns, functions)
  • Inline error highlighting from query validation
  • Multiple cursors, find & replace, code folding
  • Cmd+Enter to run query, Cmd+Shift+Enter to run selected text
  • Query history accessible via Cmd+↑/↓
B

CodeMirror 6 Editor

Lighter-weight alternative to Monaco. More customizable, smaller bundle, better mobile support (future-proofing). Used by many modern web apps (Notion, Replit).

  • ~60% smaller bundle than Monaco
  • Highly extensible — plugin system for custom behaviors
  • Same core features: syntax highlighting, autocomplete, multi-cursor
  • Better performance for very large SQL files
  • More work to match Monaco's polish out of the box

Query Editor Features (both options include)

⚡ Execution
  • Run full query or selected text
  • Multiple result tabs (run several queries)
  • Execution timer + row count
  • Cancel running queries
  • EXPLAIN ANALYZE visualization
📝 Editing
  • Schema-aware autocomplete
  • Syntax highlighting (per dialect)
  • Query formatting / prettify
  • Snippet expansion
  • Parameter placeholders
📁 Management
  • Saved queries with tags
  • Query history with search
  • Tabs for multiple queries
  • Unsaved query recovery
  • Import/export .sql files
🔒 Safety
  • Transaction mode toggle
  • Destructive query warnings (DROP, DELETE)
  • Read-only connection mode
  • Query timeout protection
  • Undo for data changes

💡 Recommendation: Option A — Monaco

Since we're already in Electron (which bundles Chromium), Monaco's larger bundle size doesn't matter — it's already part of VS Code. You get the most polished editor experience with the least custom work. Developers already know the keybindings. The SQL language service APIs make schema-aware autocomplete straightforward.
